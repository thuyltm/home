This is what we want to achieve at this phase 1:
1. TinyBit Node connect to Btcd Node
2. TinyBit Node send "version" message to Btcd Node
3. Btcd Node investigate and understand what was send
4. Btcd Node serialized and resend "version" message to Tinybit node
5. Btcd Node serialized and resend "ack" message to Tinybit node
6. Tinybit node receive what was response from Btcd Node and disconnect
